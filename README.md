# 🏎️ F1 Race Predictor: AI Strategy & Telemetry

Сервис на базе машинного обучения для симуляции и предсказания итоговых позиций гонщиков Формулы-1. Проект объединяет интеграцию с гоночной телеметрией в реальном времени, ML-алгоритмы ранжирования и современный веб-интерфейс, упакованные в Docker.

## 📌 Описание проекта
Предсказать исход гонки Формулы-1 классическими методами регрессии крайне сложно. Этот проект решает задачу иначе: алгоритм учится не "угадывать позицию", а **сравнивать гонщиков между собой** (`Learning to Rank`), учитывая их гоночный темп, погодные условия и исторические данные.

#### Основные возможности:
* **ML-симуляция гонки:** Прогноз финишного порядка (от P1 до P20).
* **Сбор телеметрии (OpenF1 & FastF1):** Автоматический парсинг скорости, процента нажатия газа/тормоза и использования DRS с тренировочных сессий.
* **Учет среды:** Интеграция погодных условий (температура трека, вероятность дождя) и результатов спринтов.
* **Интерактивный дашборд:** Встроенный веб-интерфейс в гоночной стилистике для запуска обучения и предсказаний "в один клик".
* **REST API:** Полноценный бэкенд для программного доступа к прогнозам и сырым данным.

## 🛠 Стек технологий
* **Язык:** Python 3.11

## 📊 Данные и Моделирование

### Пайплайн данных
Для формирования обучающей выборки используется многопоточный сбор данных:
1. **Jolpica API:** Положение в чемпионате (Driver/Constructor Standings), стартовая решетка (Qualifying) и результаты спринтов.
2. **OpenF1 API:** Погода во время Гран-при и покруговая телеметрия машин.
3. Данные кэшируются на жестком диске (через SQLite/FastF1 Cache) для минимизации сетевых задержек при повторных запросах.

### Архитектура модели: От Регрессии к Ранжированию
Изначально проект использовал `RandomForest` и `SVR`. Однако предсказание абсолютного числа (регрессия) плохо подходит для соревновательной среды.
Текущее ядро — алгоритм **XGBoost Ranker** (метрика `rank:ndcg`). Модель группирует входные данные по конкретным Гран-при и учится выстраивать пилотов в правильном порядке друг относительно друга, максимизируя точность угадывания именно лидеров гонки (подиума).

## 🚀 Быстрый запуск (Docker)

Проект полностью контейнеризирован. Настроен проброс томов (volumes) для сохранения обученной модели и кэша телеметрии.

1. Клонируйте репозиторий:
```bash
git clone https://github.com/Uqkgeniy/F1-race-predictor.git
cd F1-race-predictor
```

2. Запустите сборку и старт контейнеров:
```bash
docker-compose up --build -d
```

3. **Использование:**
   * 🖥 **Веб-интерфейс:** `http://localhost:8000/ui` (Главный дашборд)
   * ⚙️ **Документация API:** `http://localhost:8000/docs` (Swagger UI)

*Примечание: При первом запуске потребуется запустить процесс обучения через веб-интерфейс или API (`POST /train`), чтобы модель скачала исторические данные и сгенерировала файл `model.pkl`.*
